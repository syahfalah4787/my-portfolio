Create an immersive intro/loading experience that appears when users first enter the portfolio website.

IMPORTANT:
The intro animation MUST match the current website’s existing design system, visual identity, color palette, typography, spacing, and overall aesthetic. Do NOT create a completely different style. The transition between the intro screen and the main website should feel seamless and intentional.

The intro should feel:
- modern
- technical
- slightly cinematic
- lightweight and smooth
- interactive without being excessive
- developer-themed
- realistic, not “hacker movie cringe”

CONCEPT:
Create a startup/loading sequence inspired by:
- terminal boot logs
- code compilation
- debugging
- system initialization
- loading modules
- small glitch/error effects
- developer environment startup

The animation should give the feeling of:
“a personal developer workspace booting up.”

VISUAL DIRECTION:
- Use subtle motion and clean transitions
- Keep it minimal and elegant
- Avoid over-the-top neon cyberpunk effects
- Avoid fake “anonymous hacker” aesthetics
- Avoid loud RGB animations
- Avoid making it look like a gaming intro

IDEAS TO INCLUDE:
- Terminal-like text appearing progressively
- Loading/project initialization logs
- Small fake warnings/errors that resolve themselves
- Blinking cursor
- Smooth progress indicator
- GitHub/API/module initialization effects
- Brief code snippets or system messages
- Soft glitch transitions
- Fade into the homepage naturally

EXAMPLE TEXT STYLE:
> initializing portfolio...
> loading modules...
> fetching github repositories...
> compiling interface...
> warning resolved...
> launching workspace...

ANIMATION REQUIREMENTS:
- Keep total intro duration around 2–5 seconds
- Allow users to skip the intro
- Ensure performance remains smooth on mobile and low-end devices
- Use optimized animations only
- Prevent layout shift after loading
- Respect accessibility and reduced motion preferences

TECHNICAL REQUIREMENTS:
- Use modern frontend animation practices
- Keep the code modular and maintainable
- Ensure responsiveness across desktop and mobile
- Do not block the website for too long
- Prioritize smooth UX over flashy effects

MOST IMPORTANT:
The intro must feel like a natural extension of the current portfolio design, not a separate template pasted on top of the website.
